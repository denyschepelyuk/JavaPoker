Java Poker Project - Documentation Overview

The Java Poker Project simulates a simplified poker game, offering a modular and easily extensible design. The application leverages object-oriented principles to provide clear separation between game logic, user interface, and utility functions.

Project Structure

  • app: Contains the main application entry point. It initializes and launches the game.
  • core: Holds the primary game logic. Its sub-packages include:
    • cards: Manages playing cards and decks.
    • game: Orchestrates gameplay, handling game rounds and state management via the Table and Game classes.
    • players: Represents the human and AI players with their betting and hand management features.
  • ui: Handles the user interface. This package processes user input via the console and displays game updates.
  • util: Provides utility support. It includes classes for evaluating card hands, sorting card data, and managing game configurations.

Implementation Details

The design of the Java Poker Project is centered on a modular approach:

  • Modular Packages: Each package focuses on a specific area of responsibility, ensuring that game logic, user interaction, and utility functions are well separated.
  • Game Mechanics: The core package manages the gameplay, including shuffling and dealing cards, managing betting rounds, and determining the game winner.
  • User Interface: The ui package encapsulates the logic required for user interaction, such as input handling and game status display.
  • Utilities: The util package facilitates operations like hand evaluation and configuration management, making the core game logic more maintainable.

Usage

The application is launched via the app.App class, which contains the main method. On execution, a new game is initialized, and the game loop begins, allowing for player interaction via a command-line interface.
